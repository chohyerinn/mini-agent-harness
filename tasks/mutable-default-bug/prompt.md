# group_by_parity 버그 수정

`grouping.py`의 `group_by_parity(numbers, buckets=...)`는 입력 숫자들을
짝수/홀수로 나눠 돌려줍니다. 하지만 기본 인자로 가변 객체(딕셔너리)를 직접
써서, 함수가 정의될 때 한 번 만들어진 딕셔너리가 호출 사이에 공유됩니다.
그 결과 같은 인자로 여러 번 호출하면 이전 결과가 계속 누적됩니다.

다음을 만족하도록 고치세요.

- 매 호출은 서로 독립적이어야 함:
  `group_by_parity([1, 2])` → `{"even": [2], "odd": [1]}`,
  바로 이어서 `group_by_parity([3, 4])` → `{"even": [4], "odd": [3]}`
- `buckets`를 명시적으로 넘기면 그 딕셔너리에 그대로 담아 돌려줘야 함
  (반환값이 넘긴 객체와 동일해야 함)

`grouping.py`만 수정하세요.
